MiniZincTupleType Class

Tuples are fixed-size hetrogeneous collections. They must contain at least two elements. Unary tuples are not allowed. Tuples may contain unfixed elements.

Syntax

public class MiniZincTupleType : IMiniZincType, ZincOxide.Utils.Abstract.IGenericEquals<IMiniZincType>

Remarks

A tuple base type-inst expression tail has this syntax: tuple (ti-expr, ...). An example tuple type-instance expression is tuple (int,var float). It is a type-instance error to precede a tuple type-instance expression with var.

A tuple type is finite if all constituent elements are finite types. Otherwise false. The domain of a tuple type that is a finite type is the Cartesian pr oduct of the domains of the element types. For example the domain of tuple (1..2,{3,5}) is {(1,3),(1,5),(2,3),(2,5)}

A tuple type is varifiable if all its constituent elements are varifiable. A tuple is varified by varifying its constituent elements.

Tuples are ordered lexicographically.

A tuple variable must be initialised at instance-time if any of its constituent elements must be initialized at instance-time.

The coercions of a tuple are the following: tuple (TI1, TI2, ..., TIn) to tuple (UI1, UI2, ..., UIn) if all types TIi coerce to UIi.

In MiniZinc, tuples can only be used as array indices and must contain integers and must be fixed. Furthermore, neither tuple literals nor tupple accesses are supported.
